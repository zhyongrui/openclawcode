## Agent Error Stop-Reason Hardening

Issue `#71` exposed a workflow-fidelity bug after the successful one-line
chat-intake proof.

What went wrong:

- the builder agent returned `stopReason=error` with payload text
  `HTTP 400: Internal server error`
- `OpenClawAgentRunner` treated that as a normal text reply instead of a hard
  failure
- the builder then recorded the raw provider error as `buildResult.summary`
  even though no code changes were produced
- the workflow continued into verification and later failed with
  `Unexpected end of JSON input`
- `/occode-status` surfaced the stale build summary instead of the true
  failing-stage note because failed runs preferred `buildResult.summary`

Implementation:

- `OpenClawAgentRunner.run(...)` now throws immediately when agent output ends
  with `meta.stopReason === "error"`
- agent-backed builder and verifier runs now retry a narrow set of transient
  provider failures once before giving up:
  - `timeout`
  - `overloaded`
  - exact `HTTP 400: Internal server error`
- failed status rendering now prefers the latest `... failed:` workflow note
  over stale build summaries

Validation:

- `pnpm exec vitest run src/openclawcode/runtime/agent-runner.test.ts src/openclawcode/roles/agent-backed.test.ts src/openclawcode/testing/openclaw-plugin-integration.test.ts --pool threads`
- `pnpm exec vitest run --config vitest.openclawcode.config.mjs --pool threads`
- `pnpm build`

Outcome:

- agent provider errors can no longer masquerade as successful builder output
- chat status now reflects the true failed stage instead of stale intermediate
  summaries
- the next live proof is a rerun of issue `#71` on the updated operator build
