## Progress Update

Rebased the effort onto the real fork repository and restored the product docs into:

- `docs/openclawcode/`
- `docs/openclawcode/dev-log/2026-03-09.md`

Reintroduced the first workflow-core slice under `src/openclawcode/`:

- `contracts/` for workflow domain types and GitHub issue mapping
- `workflow/` for stage transitions and verification decisions
- `roles/` for planner, builder, verifier interfaces
- `orchestrator/` for deterministic issue-to-run sequencing
- `testing/` for fake agents and focused orchestration tests
- `index.ts` as the product export surface

Added the second slice for persistence and audit-friendly run tracking:

- expanded `WorkflowRun` with timestamps, attempt counters, stage records, and richer draft PR metadata
- added `src/openclawcode/persistence/store.ts` with a filesystem-backed `FileSystemWorkflowRunStore`
- updated the orchestrator so run state is persisted after each major transition
- added targeted persistence coverage to `src/openclawcode/testing/orchestrator.test.ts`

Added the third slice for isolated repository workspaces:

- expanded `WorkflowRun` to carry optional `WorkflowWorkspace` metadata
- added `src/openclawcode/worktree/manager.ts` with a git-backed `GitWorktreeManager`
- covered worktree preparation, reuse, changed-file collection, and cleanup in `src/openclawcode/testing/worktree.test.ts`

Added the fourth slice for executable issue workflows:

- added `src/openclawcode/github/` with GitHub repo resolution, issue fetch, draft PR creation, and merge helpers
- added `src/openclawcode/runtime/` with an embedded OpenClaw agent runner and host shell runner
- added `src/openclawcode/roles/heuristic-planner.ts` and `src/openclawcode/roles/agent-backed.ts`
- added `src/openclawcode/app/run-issue.ts` to execute the end-to-end issue workflow
- added `src/commands/openclawcode.ts` and `src/cli/program/register.code.ts` to expose `openclaw code run`
- added `src/openclawcode/testing/run-issue.test.ts` to validate publish/merge and change-request branches
- added `docs/openclawcode/mvp-runbook.md` so the workflow can be tried against this repository with a real issue

Added isolated validation entrypoints for this product slice:

- `vitest.openclawcode.config.mjs`
- `tsconfig.openclawcode.json`

Not implemented yet in this fork snapshot:

- queue persistence and adapter ingress
- human checkpoint policy beyond the current state machine
- issue suitability gating ahead of build execution
- broader automatic test selection and repo-context retrieval
