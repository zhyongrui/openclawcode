Added a runtime-hardening slice after real issue-run validation exposed sandbox/worktree edge cases:

- mirrored merged skills into generated issue workspaces so spawned agent runs see the same effective skill set as the host workspace
- normalized host-absolute workspace paths and `/workspace/...` sandbox paths back into canonical workspace-relative paths for read/write/edit tools
- forced OpenClawCode agent runs onto session-scoped sandboxes so issue worktrees no longer reuse a stale agent-scoped container mounted to the main repository
- mounted linked-worktree git metadata into sandbox containers so `git` commands inside issue worktrees can resolve both the external `.git/worktrees/...` admin dir and its absolute back-reference to the worktree path
- added focused regression coverage around workspace path guards, skill mirroring, sandbox config hashing, linked-worktree mount generation, and session-scoped sandbox config overrides
