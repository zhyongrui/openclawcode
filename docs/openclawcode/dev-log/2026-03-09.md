## Progress Update

Implemented the first code skeleton:

- `package.json` and `tsconfig.json`
- `src/contracts/` for core workflow types
- `src/workflow/state-machine.ts` for stage transitions
- `src/roles/interfaces.ts` for planner, builder, verifier contracts
- `src/index.ts` export surface

Implemented the next integration layer:

- `src/orchestrator/` for workflow execution sequencing
- `src/contracts/github.ts` for GitHub issue payload mapping
- `src/testing/` fake agents and orchestrator tests
- `docs/specs.md` for a stable written contract

Implemented workflow persistence and richer run tracking:

- expanded `WorkflowRun` with timestamps, attempt counters, stage records, and draft PR state
- expanded workflow stage model to include `draft-pr-opened`, `changes-requested`, `ready-for-human-review`, and `escalated`
- added `src/persistence/store.ts` with a filesystem-backed `WorkflowRunStore`
- updated orchestrator persistence hooks so state snapshots are saved after each major transition
- added persistence-focused tests and updated orchestrator tests to cover the richer state model

Implemented isolated repository workspace support:

- added `src/worktree/` with a `WorkflowWorkspaceManager` contract and `GitWorktreeManager`
- extended `WorkflowRun` with a persistent workspace lease model
- updated the orchestrator so build execution can prepare a worktree before handing control to the builder role
- added real git-backed tests for worktree creation, reuse, changed-file listing, and cleanup

Moved the OpenClaw plugin adapter plan into the main repository:

- added `src/integrations/openclaw-plugin/` as the OpenClaw-facing adapter layer
- ported queue ingress, queue persistence, and queue-drain service code into `openclawcode`
- added tests so the plugin adapter can be validated without depending on a modified OpenClaw checkout
- documented the “single repo, two layers” structure in `docs/openclaw-plugin-integration.md`

Updated repository strategy documentation for the selected fork-based path:

- revised the strategy docs to reflect `openclawcode` as the main OpenClaw-derived product repository
- clarified that OpenClaw adapter code remains in-repo instead of living in a third repository
- added `docs/upstream-sync-policy.md` to define sync cadence and conflict-reduction rules
