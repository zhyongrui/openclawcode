## Slow Integration Timeout Hardening

The first full-suite rerun after `workspaceRepoRoot` landed exposed three
time-based false reds on this host even though the targeted behavior was
correct:

- `src/openclawcode/testing/openclaw-plugin-store.test.ts`
  - `persists pending approvals and consumes them when approved`
  - `persists queue entries and statuses across store instances`
- `src/openclawcode/testing/run-issue.test.ts`
  - `publishes and merges when verification approves and merge is enabled`

Field note:

- each test passed in isolation, but each single-test run still took about
  19 seconds wall clock because transform/import time dominated on this host
- the default `testTimeout = 30000` in `vitest.openclawcode.config.mjs` left
  very little headroom for the serial full-suite run
- fix chosen:
  - keep the global timeout unchanged
  - raise only those three slow integration tests to `60_000`

This keeps genuine hangs visible while preventing fresh sessions from
misclassifying host-speed variance as a functional regression.
